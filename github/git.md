# Git

Gitは、**分散型**バージョン管理システムと呼ばれるものです。これは、ファイルの変更履歴を記録し、複数人で同時に同じファイルを編集する際に、変更内容を安全に管理するためのツールです。

## バージョン管理システム

- **変更履歴の追跡:** ファイルの変更履歴を記録することで、過去の状態に戻ったり、変更内容を比較したりすることができます。
- **共同作業の効率化:** 複数人で同時に同じファイルを編集する際に、変更内容の衝突を防ぎ、スムーズな共同作業を可能にします。
- **バックアップ:** ファイルの変更履歴を保存することで、誤ってファイルを削除した場合でも、過去の状態に復元することができます。

## 集中型

![集中型](https://storage.googleapis.com/zenn-user-upload/622818b17001-20230503.jpg)
集中型バージョン管理システムでは、リポジトリ（ファイルの変更履歴を保存する場所）は中央サーバーに一つだけ存在します。開発者は、中央サーバーからファイルをチェックアウトして編集し、変更を中央サーバーにチェックインします。

## 分散型

![分散型](https://storage.googleapis.com/zenn-user-upload/f44aed454a0e-20230503.jpg)

集中型バージョン管理システムとは異なり、Gitでは各開発者のPCにリポジトリ（ファイルの変更履歴を保存する場所）の完全なコピーが存在します。これにより、ネットワークに接続していなくても作業を続けることができ、また、中央サーバーに障害が発生した場合でも、他の開発者のリポジトリから復旧することが可能です。

## Gitホスティングサービス

Gitホスティングサービスは、Gitリポジトリをインターネット上で共有・管理するためのサービスです。これらのサービスを利用することで、複数人での共同開発が容易になり、プロジェクトのバックアップや公開も簡単に行えます。

代表的なGitホスティングサービスには以下のようなものがあります。

| サービス名 | URL | 説明 |
|------------|-----|------|
| GitHub | [https://github.co.jp/](https://github.co.jp/) | 世界で最も利用されているGitホスティングサービスの一つで、オープンソースプロジェクトから企業での開発まで幅広く利用されています。 |
| GitLab | [https://about.gitlab.com/ja-jp/](https://about.gitlab.com/ja-jp/) | GitHubと同様の機能を提供し、オンプレミスでの運用も可能です。CI/CD（継続的インテグレーション/継続的デリバリー）機能も充実しています。 |
| BitBucket | [https://bitbucket.org/product/ja](https://bitbucket.org/product/ja) | Atlassian社が提供するサービスで、Jiraなどの同社の製品との連携が容易です。 |

これらのサービスを利用することで、Gitの利便性を最大限に活用し、効率的な開発を行うことができます。
